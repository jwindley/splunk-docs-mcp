**Core:**

- `pydantic` — used via `mcp` for tool parameter validation and `Field` descriptors in `server.py`

**HTTP:**
- `httpx` 0.28.1 — async HTTP client (`AsyncClient`) for crawler in `crawler.py`; sync client for setup downloads in `setup.py`

**HTML Extraction:**
- `trafilatura` 2.0.0 — primary HTML→Markdown extractor (text-density heuristics); used in `extractor.py`
- `beautifulsoup4` 4.14.3 — fallback extractor for pure-table and index pages; HTML parsing in `extractor.py` and link extraction in `crawler.py`
- `lxml` 6.0.4 — fast XML/HTML parser; used as the BS4 parser backend
- `markdownify` 1.2.2 — converts BS4 parse trees to Markdown in the fallback extraction path in `extractor.py`

**Embeddings / ML:**
- `sentence-transformers` 5.4.1 — loads and runs `all-MiniLM-L6-v2` (384-dim embeddings) in `server.py` and `cli.py`; depends on PyTorch
- `numpy` 2.4.4 — in-process cosine similarity via dot product on embedding matrix in `server.py`

**Testing:**
- `pytest` 9.0.3 — test runner; configured via `[tool.pytest.ini_options]` in `pyproject.toml`
- `pytest-asyncio` 1.3.0 — async test support; `asyncio_mode = "auto"` so no per-test decorator needed

**Build/Dev:**
- `hatchling` — build backend (`[build-system]` in `pyproject.toml`); builds the wheel from `src/splunk_docs_mcp/`

## Key Dependencies

**Critical:**
- `mcp` 1.27.0 — exposes the entire tool surface to Claude; all tool definitions in `server.py` are decorated with `@mcp.tool()`
- `sentence-transformers` 5.4.1 — semantic search capability; model loaded eagerly at server startup for zero cold-start penalty on first call
- `trafilatura` 2.0.0 — primary content extraction; fallback to `markdownify`+`beautifulsoup4` when trafilatura returns <100 chars
- `httpx` 0.28.1 — both async (crawler) and sync (setup downloader) HTTP

**Infrastructure:**
- `numpy` 2.4.4 — entire semantic search ranking computed in-process as matrix dot product; no vector DB dependency
- `lxml` 6.0.4 — required by BS4 for HTML parsing; also used by trafilatura internally

## Database

**Engine:** SQLite (stdlib `sqlite3` module)

**Mode:** WAL (`PRAGMA journal_mode=WAL`) — allows concurrent reads during crawler writes

**Search:** FTS5 virtual table with `porter unicode61` tokenizer; content table pattern (`content=documents, content_rowid=id`) with INSERT/UPDATE/DELETE triggers to keep FTS in sync without duplicating text

**BM25 tuning:** `bm25(documents_fts, 10.0, 1.0)` weights title matches 10× over body; lower score = better match (SQLite convention)

**Embeddings storage:** 384-dim float32 BLOBs in `documents.embedding` column; loaded entirely into a NumPy matrix at server startup

**File:** `data/splunk_docs.db` (gitignored; regenerated by crawl or downloaded via `splunk-setup`)

## Notable stdlib Modules

- `asyncio` — BFS crawler concurrency (`asyncio.Queue` + N worker tasks + `queue.join()` in `crawler.py`)
- `sqlite3` — all DB access; single module-level connection singleton in `server.py`
- `hashlib` — SHA-256 content hashing for incremental re-crawl skip and Option B dedup (`db.py`)
- `pathlib` — all file path construction throughout (`config.py`, `cli.py`, `setup.py`)
- `dataclasses` — `CrawlSource` definition in `config.py`
- `argparse` — CLI argument parsing in `cli.py`, `setup.py`, `merge.py`
- `json` — `version_tags` JSON array serialisation/deserialisation in `db.py`
- `logging` — structured stderr logging in `server.py` (avoids contaminating stdio MCP protocol)
